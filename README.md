# Folio, a demo store that runs AnnotateQA on itself

Folio is a tiny storefront we use to show [AnnotateQA](https://annotateqa.com)
doing the thing it is for: turning a customer's bug report into a reviewed pull
request, live.

The store is a static web app (everything under `public/`), it has the
AnnotateQA widget installed, and it ships with one real, planted bug. Clicking
**"Apply newsletter discount (SAVE10)"** crashes the cart: the total stays
$50.00 and the browser console throws a `TypeError`, because `savingsLabel()`
in `public/src/pricing.js` reads `promo.amount` when the promo's discount field
is named `value`.

A customer reports it through the widget. AnnotateQA reproduces it by driving
the running store with Playwright (it clicks Apply and catches the console
error), writes the fix, verifies it against this repo's existing test suite,
then opens a **draft pull request** you review and merge.

**The bug is planted. The pull request is real**, generated by the hosted
Auto-Fix loop end to end:

- Cart crashes on "Apply SAVE10" → [PR #4](https://github.com/ibrolord/folio-shop/pull/4)

## Run it

```bash
npm install
npm test        # the existing suite, green, because the bug sits in an untested path
npm run dev     # serve the store at http://localhost:3000
```

## Layout

```
public/index.html      the store (runs the widget + the cart)
public/src/pricing.js  cart + promo math (the planted bug lives here)
public/src/...         email, catalog, format
test/                  the existing suite AnnotateQA verifies fixes against
```

The reproduction is written by AnnotateQA from the captured report, not from
this repo. Details in [BUGS.md](./BUGS.md).
